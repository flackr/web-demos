Introduction

This demonstrates potential accessibility patterns for carousels using aria-flowto as well as implicit tab orders

tabindex

Rather than setting inert on offscreen content, tabindex is set to -1 on all focusable elements in offscreen tabpanels to ensure that the tab order skips the rest of the content.

aria-flowto

In order to provide screen reader users with a sensible reading order, the aria-flowto property indicates that flow should return to the tab controls.

<div role="tabpanel" id="slide3" aria-flowto="tab3">

aria-controls

The aria-controls property indicates that the scroll marker group and scroll buttons control the scrolling container.

<div class="scroll-marker-group" aria-controls="scroller">
  <div class="scroll-marker"></div>
  ...
</scroll-marker-group>
<button aria-controls="scroller">
  Previous slide
</button>
<button aria-controls="scroller">
  Next slide
</button>

aria-details

The aria-details property links from the tab to the associated tabpanel.
